# EXP-050: NLP Parser v3 — Bug Fixes + New Filters

**Date:** 2026-04-01
**Skill:** job-tracking
**Component:** Korean NLP Query Parser

## Summary

Fixed 3 bugs and added 5 missing filter features in the Korean NLP query parser for job tracking.

## Bugs Fixed

### 1. SQL Quote Bug (Critical)
The keyword LIKE filter was generating unbalanced SQL:
```sql
-- Before (broken):
(j.title LIKE '%백엔드%' OR j.company LIKE '%백엔드%)
--                                         missing quote ^
-- After (fixed):
(j.title LIKE '%백엔드%' OR j.company LIKE '%백엔드%')
```
This would cause SQL syntax errors at runtime for any query with an unknown Korean keyword.

### 2. Company Substring Conflict
Query "카카오뱅크 공고" matched both `카카오` and `카카오뱅크`, producing:
```sql
j.company LIKE '%카카오%' AND j.company LIKE '%카카오뱅크%'
```
Fix: Sort companies by length (longest first) and skip if already consumed by a longer match.

### 3. 최신순 Keyword Spill
"지원한 거 최신순" treated `최신순` as a title keyword filter instead of recognizing it as a sort order. Fix: Consume sorting keywords early before keyword extraction.

## New Features

| Feature | Keywords | SQL Filter |
|---|---|---|
| Salary | 연봉, 급여, 연수입 | `j.salary IS NOT NULL AND j.salary != ''` |
| Deadline urgency | 마감임박, 곧마감 | deadline within 7 days |
| Today's deadline | 오늘 마감 | deadline = today |
| Tomorrow's deadline | 내일 마감 | deadline = tomorrow |
| N days left | 3일 남은 | deadline within N days |
| Has deadline | 기한 있는, 데드라인 있는 | `j.deadline IS NOT NULL` |
| Deadline sort | 마감순, 마감 빠른순 | `ORDER BY j.deadline ASC` |
