# EXP-153: MCP Server Keyword Search, Delete Job, Save Application Tools

**Date:** 2026-04-07
**Skill:** mcp-server
**Metric:** mcp_tool_completeness

## Hypothesis

MCP server was missing three basic capabilities: (1) keyword search on title/company — the primary query pattern generated by the NLP parser, (2) job deletion with cascade to related records, and (3) explicit application creation/upsert. Without keyword search, MCP-based workflows couldn't perform the most common query type (e.g., "프론트엔드 공고" or "카카오").

## Change

1. Added `keyword` filter to `db_search_jobs` with `title LIKE ? OR company LIKE ?`
2. Added `keyword` to `JobFilters` interface
3. Added `db_delete_job` tool that cascades: matches → applications → job
4. Added `db_save_application` tool that upserts by job_id (creates or updates)
5. Fixed `test_title_career_stage_korean.js` stale expectation: `신입-5년` expected `junior` but EXP-151 changed it to `mid`

## Impact

MCP-based workflows (e.g., Claude/Codex using the job-hunter MCP server) can now:
- Search jobs by keyword: "프론트엔드", "카카오", "Data Scientist"
- Delete jobs with proper cleanup
- Create/update applications explicitly without relying on auto-creation on job save
